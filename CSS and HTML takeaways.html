Similarities and differences between HTML and Markdown

I would say the biggest similarity between HTML and Markdown is that they both use code to format documents, but this is a very obvious observation. Another more notable similarity between the two is that both are compatible in GitHub. This is very convenient because even though it's beneficial to know how to use different code editors and developers, it's nice to only have to use one. That way, you can store different types of documents in the same place, making them easier to find. HTML and Markdown have some distinct differences, however. The biggest one being their code. Markdown is a lightweight markup language, meaning it uses plain text formatting that is purposefully simple enough for most to understand. It is very easy to learn, whereas HTML is a bit more difficult because of how many elements there are to learn. Another difference between the two is that writers have more creative freedom with their HTML documents than with their Markdown documents. This is related to my previous point because although HTML is more difficult to learn, it correlated directly with how much freedom you have over the style of the document. Users can do a lot more with their documents' styles in HTML, whereas Markdown's simplicity makes it difficult implement different styles.
